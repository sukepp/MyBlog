The software architecture of PX4

The PX4 autopilot software can be divided into three main parts: real-time operating system, middleware and flight control stack.

1.Nuttx real-time operating system:

It Provides a POSIX-style user operating environment for underlying task scheduling....

2. PX4 middleware:

The PX4 middleware runs on top of the operating system and provides device drivers and a micro object request broker (uORB) for asynchronous communication between individual tasks running on the pilot. My understanding is that it can be seen as an interprocess communication mechanism.

3. PX4 flight control stack:

The flight control stack can use the PX4 control software stack or other control software such as APM: Plane, APM: Copter, but must run on the PX4 middleware.

This stack can be divided into 4 parts:

Name Definition
Navigation part The mode of operation and the mission are determined based on the current flight status of the aircraft and the user commands received.
Position and posture estimation part Get the position and attitude information through the sensor.
Position and attitude control part The flight state is controlled to achieve the desired position and attitude based on the desired deviation from the current.
Controller output part Mixer and actuator, pwm limiter.
